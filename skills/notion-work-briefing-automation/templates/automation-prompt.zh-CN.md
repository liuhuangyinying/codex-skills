使用 Notion 连接器生成每日工作早报。用户只需要看到最终结果，不要在正文中暴露检索过程、权限限制、生成说明、补充说明或“自动生成”等过程性文字。若需要更新已有早报，必须重构为一份完整成稿，替换旧内容，不要用“补充”的形式追加。

主要数据来源：
1. Notion「任务面板」：{{TASK_DATABASE_URL}}，数据源 {{TASK_DATA_SOURCE_ID}}。核心字段包括：Task name、Assignee、Status、需求传递日期、计划完成日期、项目、目标。
2. Notion「工作记录/知识库页面」：{{WORK_LOG_SOURCE_URL}}，如有数据源则使用 {{WORK_LOG_DATA_SOURCE_ID}}。不要只依赖任务面板；很多真实工作记录可能只写在工作记录页面里。

输出位置：在 Notion 页面「工作日志」下新建或更新子页面：{{OUTPUT_PARENT_PAGE_URL}}。页面标题必须是当天日期，格式为 yyyy-MM-dd 「工作早报」，例如 2026-06-23 「工作早报」。

执行步骤：
1. 以当前执行日期为“今天”，计算“昨天”和“明天”。早报需要读取并总结最近两天的工作记录：昨天 + 今天；如果是补某一天的早报，则读取该日期和前一天的工作记录。
2. 从「任务面板」整理：
   - 今天应该做什么：优先列出计划完成日期为今天、需求传递日期为今天、状态未完成或进行中的任务。
   - 明天要做什么：列出计划完成日期为明天或需求传递日期为明天的任务。
   - 昨天团队做了什么：按 {{CORE_MEMBERS}} 分别总结；如有其他负责人也可放在“其他成员”中。
3. 从「工作记录/知识库页面」补充读取最近两天工作记录：
   - 在 {{WORK_LOG_SOURCE_URL}} 内按日期关键词搜索，例如 yyyy-MM-dd、M月d日、MM-DD、昨天/今天相关标题。
   - 重点读取最近两天创建或更新、标题或正文含日期的页面。
   - fetch 命中的工作记录页，提炼为“实际完成/推进事项”“沉淀经验”“新增待办/风险”。
   - 工作记录优先用于补充任务面板没有覆盖的真实工作，不能因为任务面板无记录就写“没有进展”。
4. 如果 Notion 的 data source SQL 查询可用，优先使用 query_data_sources 精确查询；如果因为权限限制不可用，就使用 Notion search 限定 page_url 或 data_source_url，分别搜索日期、人员姓名、状态关键词和任务关键词，再 fetch 相关页面进行归纳。
5. 新建或更新页面内容使用中文，结构如下，正文只呈现结果：

# 今日重点
用 3-6 条概括今天最需要推进的事项。

# 今日应该做什么
按负责人或事项分组列出任务，包含任务名、状态、日期、项目/目标；高优先级或临近截止放前面。

# 明天要做什么
按负责人或事项分组列出任务，包含同样字段。

# 近两天工作提炼
综合「任务面板」和「工作记录/知识库页面」记录，提炼昨天 + 今天实际推进了什么、沉淀了什么、新增了什么。

# 昨天团队进展
必须分别列出 {{CORE_MEMBERS}}；每个人下方用简短 bullet 总结完成/推进了什么。没有明确记录时写“未找到明确记录”，不要编造。

# 风险与提醒
列出逾期、无人负责、状态停滞、信息不完整、新增待确认的任务。

# 数据来源
列出使用到的任务页面和工作记录页面链接。

6. 写入 Notion 前检查同名页面是否已经存在于「工作日志」下；如果已存在，则将页面内容重构为完整最终版并替换旧内容，避免重复创建，也不要追加“补充”。若无法可靠检查是否存在，则创建页面。
